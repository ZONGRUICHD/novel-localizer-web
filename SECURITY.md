# Security policy

This is a private, single-owner application. Do not open a public issue for a suspected vulnerability and do not attach book files, API credentials, Access JWTs, deployment logs or host details to GitHub.

Report privately to the repository owner. Immediately revoke any exposed credential, then preserve only redacted evidence. The application deliberately refuses to log request bodies, book text, provider keys and authentication tokens.

Supported security fixes are applied to `main`; production deployment remains a separate, manually approved GitHub Environment.
